Python Project: Image Classification

1. Requirement

You are expected to write a python project that learns a model for image classification. The project uses a Minist dataset, which is a database of handwritten digits has a training set of 60,000 examples, and a test set of 10,000 examples. This project is based on python and PyTorch (an open-source machine learning framework). For the ease of implementation, the model architecture, loss functon and data loader have been provided [here]. Please install the python package by following the instruction there. Students should implement the basic features including:

  • Training Function (10%): Please implement the "train" function. The train function should use training data (consisted of images-label pairs) to update the model parameters. The training loss should be record by a file (e.g., .txt file) after each update. The training loss and accuracy should be print after each update.
  • Testing Function (10%): Please implement the "test" function. The test function should use the testing data (consisted of only images). The testing loss and accuracy should be record by a file (e.g., .txt file) after each update.
  • Plotting Function (20%): Please implement the "plot" function. The plot function should generate the line charts based on the records training loss, testing loss and testing accuracy (i.e., generate three plots) for each run. The plot should include the title and labels.
  • Random Seed (20%): Please include a random seed for each run. The DataLoader should use this random seed for generating data. The training and testing results (accuracy and loss) should be recorded for each random seed. Please use three random seeds (123, 321, 666) and implement three runs (one random seed for each run). You can modify the random seed in the config file in "./config/minist.yaml" of your project. Please use the recorded results to plot the mean of training loss, testing loss and testing accuracy.
  • Python Multi-Processing (30%): Please improve the program by adding Python multi-processing into the code. The multi-processing should increase the efficiency of running by using one random seed for each process (use one run with three process instead of three runs), although it might take more resource.
  • Documentation (10%): Please write a documentation explaining how do you implement the above functions.

2. Submission

Please submit the 1) generated plots (12 plots, 3*3 for three runs and 3 for the mean results), 2) python code and 3) documentation by compressing them into a zip file.
