Java Chat Room

1. Basic Features

You are expected to write a small chat application with Java. A basic application must have the following features:

  • Multi-Client Chat(20%). The application should have both a server and a client implementation to maintain a chat room and allow multiple users to chat together.
  • Login System(20%). The program should have a login window (Note that a registration system is not necessary. See the advanced features listed below). The profiles (e.g., age, gender, and address) and their account information (e.g., usernames and passwords) must be recorded in a database. Every time a user logins, the program should check whether his or her account exist and if the input password matches records in the dataset.
  • Java GUI (20%). The program should have a Graphical User Interface (GUI). This interface should be user-friendly, with buttons, text fields, and other features to maintain a comfortable experience. The command interface is not allowed.
  • Documentation, Presentation, and Code (20%). 1) Please write a tutorial about the main features in your project, including how do you implement them and how to run your code. Please make sure TAs can run your code on their machine. (10%) 2) Please prepare a presentation for your project by recording a video. The video should record the screen while your code is running on your laptop or desktop (5%). 3) Please prepare your code for submission. The main functions and classes in the code must have comments in a reader-friendly format. (5%)

2. Advanced Features

Note that you can earn 80% of your scores by submitting the project with the above basic features. That is to say, to get a higher score, you may want to consider adding some advanced features ( Please pick some features from the followings. The corresponding bonus scores will be added to your final score, and the upper bound of your final score is 100%. That's to say, you can get at most 100% by picking four 5% features or one 10% with two 5% features. ):

  • Registration system (5%). new users can register in this system. The information must be written to the dataset so that new user can login in.
  • Emoji (5%): the chat box should allow the users to send emojis. The list of emojis can be pre-loaded into the application or uploaded by users..
  • Sending Pictures (5%). The chat box should allow the users to send pictures. The pictures can be uploaded by users.
  • Sending Videos(5%). The chat box should allow the users to send voice messages. This voice can be pre-recorded (on the local machine) and collected online.
  • Animation (5%). The chat window, login window, and registration window should have some background animations so that the interface looks fancier..
  • Voice Chat (5%). The program supports voice chat in the real time among multiple users.
  • Message History (5%). Client can storage the chat message. The previously stored message can be resumed when the same client login the other time. The message should be store in the server (e.g., Database).
  • Editing History (5%). Client can edit the message sent within 2 minutes. The editing includes retreating, changing and quoting the text message. All other clients must be aware of this edit.
  • Web Application (10%-15%). transfer the local system to a web application with basic web tools or more updated tools. These tools include Java servlet (10%), JSP (10%), Struts (10%), Spring (10%) or SpringBoot (15%).
  • Other cool features (5%): if you have other cool ideas for improving your project, please let me know as early as possible (Before Feb. 28th, 2023) and I will consider some extra top-ups.
